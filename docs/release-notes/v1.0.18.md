# ConfigPilot v1.0.18

本版本修复 Claude Desktop 第三方推理网关地址被错误追加 `/v1` 的问题，避免 Claude Desktop 自行请求 `/v1/messages` 时产生重复版本路径并拒绝连接测试。

## 主要更新

- Claude Gateway endpoint 通过完整 HTTP(S) URL 校验后原样写入，不再复用 Codex 的 `/v1` 地址归一化逻辑。
- Codex `base_url` 缺少 `/v1` 时仍会自动补全，Codex 行为不受此次修复影响。
- Claude 页面明确提示 endpoint 会原样写入，避免把 Codex 与 Anthropic 两种地址规则混淆。
- 增加回归测试，确保 `https://api.9li.life` 写入 Claude 配置后仍保持原值。

## 升级说明

ConfigPilot 已包含应用内更新器。Windows 用户可在应用内完成下载、静默安装和重启；macOS 用户检测到新版后会打开官方 Release 页面。

旧版本已经写入配置文件的 `/v1` 不会被自动删除，因为部分供应商可能确实使用自定义子路径。升级后请在 Claude 页面把网关地址改回供应商提供的根地址并重新点击一次“应用”，然后完全退出并重新打开 Claude Desktop。

例如，使用 `api.9li.life` 时应填写：

```text
https://api.9li.life
```

## macOS 首次安装须知

当前 macOS 版本仅支持 Apple Silicon（M 系列芯片），不支持 Intel Mac。

维护者目前没有 Apple Developer Program 会员，因此 DMG 尚未使用 Developer ID 签名和 Apple 公证。macOS Gatekeeper 可能显示“ConfigPilot 已损坏，无法打开”，但这不代表 DMG 文件真的损坏。

请先把 `ConfigPilot` 拖入 `Applications`，再打开“终端”执行：

```bash
xattr -dr com.apple.quarantine "/Applications/ConfigPilot.app"
open "/Applications/ConfigPilot.app"
```

同一已安装版本通常只需执行一次。请仅从本项目官方 Releases 下载。
