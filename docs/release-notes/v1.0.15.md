# ConfigPilot v1.0.15

本版本将界面引擎升级至 PrismQML 0.3.1.1，并完成配置、模型请求、安装与更新下载路径的异步化收口，避免耗时文件、网络和子进程操作阻塞界面。

## 主要更新

- PrismQML 从 `0.2.24.13` 升级至 `0.3.1.1`，获得异步页面加载、导航滚动、图标与按钮按需加载，以及 QML 生命周期稳定性改进。
- Claude Desktop 与 Codex 的配置读取、写入、模型目录刷新和模型接口请求移出 GUI 主线程。
- Codex 密钥保存与模型请求改为同一配置顺序读取，避免保存新密钥后立即请求仍使用旧密钥。
- `config.toml` 与 `auth.json` 使用原子替换；损坏的认证文件不再阻止正常配置加载。
- Claude 安装下载、校验和取消流程增加销毁保护，取消后不会再启动已经排队的安装程序。
- 更新安装包在后台限流写入唯一临时文件，应用退出时会停止网络请求、排空或清理后台任务。
- 后台配置写入期间锁定相关编辑区，避免异步回填覆盖用户的新草稿。

## 升级说明

ConfigPilot 已包含应用内更新器。Windows 用户可在应用内完成下载、静默安装和重启；macOS 用户检测到新版后会打开官方 Release 页面。

本版本更新了内置 PrismQML 引擎，必须安装 `v1.0.15` 的新程序包；仅更新配置文件不会获得新的界面引擎。

## macOS 首次安装须知

当前 macOS 版本仅支持 Apple Silicon（M 系列芯片），不支持 Intel Mac。

维护者目前没有 Apple Developer Program 会员，因此 DMG 尚未使用 Developer ID 签名和 Apple 公证。macOS Gatekeeper 可能显示“ConfigPilot 已损坏，无法打开”，但这不代表 DMG 文件真的损坏。

请先把 `ConfigPilot` 拖入 `Applications`，再打开“终端”执行：

```bash
xattr -dr com.apple.quarantine "/Applications/ConfigPilot.app"
open "/Applications/ConfigPilot.app"
```

同一已安装版本通常只需执行一次。请仅从本项目官方 Releases 下载。
