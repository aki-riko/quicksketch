# ConfigPilot v1.0.17

本版本新增安全的“恢复初始设置”功能，只撤销 ConfigPilot 实际写入且此后未被外部修改的 Codex 配置，不会把工作区设置回滚到旧备份。

## 主要更新

- Codex 页面新增“恢复初始设置”入口和二次确认，恢复完成后会明确提示恢复项与跳过项数量。
- 为每个 ConfigPilot 管理字段记录原值和最后写入值；只有当前值仍等于最后写入值时才恢复，用户在外部做过的新修改会原样保留。
- 恢复始终以当前 `config.toml` 为主体，仅定点处理连接、模型、思考等级、上下文和 provider 字段。
- `[projects.*]` 工作区信任设置以及 `notify`、`features` 等其它 Codex 配置不会从旧备份恢复。
- API Key 使用相同的所有权校验；外部更新过的 `auth.json` 不会被覆盖，ConfigPilot 新建的认证文件则可以安全撤销。
- 恢复记录采用用户目录内的原子写入文件；旧版本没有字段级记录时不会猜测或整份覆盖配置。

## 升级说明

ConfigPilot 已包含应用内更新器。Windows 用户可在应用内完成下载、静默安装和重启；macOS 用户检测到新版后会打开官方 Release 页面。

“恢复初始设置”从安装 `v1.0.17` 后第一次实际应用配置或保存 API Key 时开始记录。旧版本遗留改动没有可靠的字段级基线，因此不会进行猜测性恢复。

## macOS 首次安装须知

当前 macOS 版本仅支持 Apple Silicon（M 系列芯片），不支持 Intel Mac。

维护者目前没有 Apple Developer Program 会员，因此 DMG 尚未使用 Developer ID 签名和 Apple 公证。macOS Gatekeeper 可能显示“ConfigPilot 已损坏，无法打开”，但这不代表 DMG 文件真的损坏。

请先把 `ConfigPilot` 拖入 `Applications`，再打开“终端”执行：

```bash
xattr -dr com.apple.quarantine "/Applications/ConfigPilot.app"
open "/Applications/ConfigPilot.app"
```

同一已安装版本通常只需执行一次。请仅从本项目官方 Releases 下载。
